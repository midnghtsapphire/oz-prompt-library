# OZ Prompt Library

A centralized, organized collection of high-quality prompts for all projects and workflows.

## Purpose

This library serves as your single source of truth for prompts across all Mechatronopolis projects. Instead of having prompts scattered across documents, notes, and messages, everything lives here with version control and easy searchability.

## Structure

### 📁 Folders

- **`prompt-engineering/`** - Meta prompts, prompt creation, and prompt optimization
- **`app-development/`** - App ideation, blueprints, planning, and architecture
- **`marketing/`** - SEO, content creation, social media, and copywriting
- **`automation/`** - Workflow automation, agent development, and tool-calling
- **`data-analysis/`** - Data processing, analysis, visualization, and insights
- **`research/`** - Research methodologies, investigation, and fact-finding
- **`accessibility/`** - Alt text, WCAG compliance, and inclusive design
- **`business-strategy/`** - Strategy, planning, monetization, and growth

## How to Use

### Adding New Prompts
1. Choose the appropriate folder
2. Create a new `.md` file with a descriptive kebab-case name
3. Use the prompt template (see `TEMPLATE.md`)
4. Commit and push to GitHub

### Finding Prompts
- Browse folders by category
- Use GitHub search for keywords
- Check the index below for quick reference

### Modifying Prompts
- Edit the file directly
- Test the modified prompt
- Document changes in commit message
- Consider versioning for major changes (e.g., `blue-ocean-v2.md`)

## Prompt Template

Every prompt should include:
- **Title** - Clear, descriptive name
- **Purpose** - What this prompt does
- **Use Cases** - When to use it
- **Instructions** - The actual prompt
- **Variables** - Any customizable parameters
- **Examples** - Sample outputs (optional)
- **Notes** - Tips, warnings, or context

See `TEMPLATE.md` for the full template.

## Integration with Workflow

### From Brain Dump to Prompt Library
1. Capture prompt ideas in `/projects/braindump/`
2. During daily processing, identify prompt-worthy items
3. Add to prompt library with proper formatting
4. Reference in `/projects/tasks/` if further development needed

### Using Prompts in Projects
- Copy prompt from library
- Customize variables for your use case
- Document which prompt you used in project notes
- Submit improvements back to library
